# 开发历程

这个项目最初来自同事的提议，他们希望可以批量抓取数据，避免客户需要检测数据的时候花费大量人力去抄录。特别是对于新开发的项目，方便通过汇总数据监管过程。项目于 2025 年 1 月中开始构思（距离春节还有十几天），于春节期间完成第一版，上班后就投入生产环境测试使用。

2026.6.8-2026.6.19  
我负责的一个箱体产品项目存在大量检测尺寸，报告有十几页，按客户要求在执行全检。  
本工具在实际工作中效率低下，每次导出时间稳定在 25-30s，我尝试优化也难以提升。  
我在研究中发现通过 PC-DMIS 内置 BASIC 执行效率非常高，至少相比外部调用有百倍的性能差异。  
我就开始学习 BASIC，并准备重构项目。开始重构项目的时候刚好遇到领导安排我出差，就这个箱体产品去客户工厂现场跟踪装配情况和商谈一些事务，从周二去到周六下午才回来。  
BASIC 脚本部分在出差期间完成了，回来后停了好几天都没动了，直到 19 号放端午节我又开动了，把 Python 部分重构完成，整体可以联动工作。  

要学习 PC-DMIS 的 BASIC，可以参考[我的博客记录](https://blog.iyatt.com/?p=24885)。
